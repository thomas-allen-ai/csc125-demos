Understanding the Solver Comparison

← Back to solver

The Big Picture

Both algorithms find the same solution (the shortest path). The difference is how much work they do to find it.

A* uses a heuristic (educated guess) to search smarter. BFS searches blindly in all directions.

The Two Algorithms

A* (Informed)

Uses Manhattan distance to prioritize moves that seem to get closer to the goal.

Explores promising paths first.

BFS (Uninformed)

Explores all states at distance 1, then distance 2, then distance 3...

No preference—tries everything equally.

What the Metrics Mean

Nodes explored
How many puzzle states the algorithm examined. Lower is better—it means less work.
Peak frontier
The largest number of states waiting to be explored at any moment. This reflects memory usage. BFS often has huge frontiers; A* keeps them smaller.
Solution length
Number of moves in the solution. Both algorithms find optimal (shortest) solutions, so this should match.
Time
Milliseconds to find the solution. Affected by nodes explored and JavaScript overhead.

Why A* Wins

A* uses the Manhattan distance heuristic: for each tile, how many squares away is it from its goal position? States with lower totals are explored first.

This means A* doesn't waste time on paths that are obviously going the wrong direction.

Example: At scramble depth 12, BFS might explore 30,000+ states while A* explores ~100. Same solution. Wildly different cost.

The Node Limit

The demo stops after 100,000 nodes to prevent your browser from freezing. For deep scrambles, BFS may hit this limit and give up, while A* still finds the solution easily.

This isn't a flaw in BFS—it's demonstrating why heuristics matter. Without smart prioritization, search becomes impractical.
